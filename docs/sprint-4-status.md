# Sprint 4 状态

日期：2026-05-29

## 目标

Sprint 4 的目标是把练习页从“有入口”推进到“能形成真实输出闭环”。重点是让输入材料经过跟读、复述、短写作之后，能够产生记录、反馈和后续复习资产。

## 本轮范围

- 复述训练真实入口
- 复述关键点反馈
- 复述练习记录
- 后续录音复述
- 后续 AI 复述反馈
- 后续角色扮演闭环

## 已完成

### 0. ChatGPT 风格色调调整

- 全局背景切换为白色和浅灰白组合
- 面板、卡片、边框、徽标和选中态切换为更低饱和的中性色
- 主按钮、Tab、复选框、开关和进度条切换为黑灰高对比风格
- 首页、学习页、练习页、复习页和设置页的大面积绿色高亮已改为中性工作台风格
- 绿色从界面和 PWA 图标中移除，不再作为品牌色或主要交互色
- PWA theme color 和图标同步为黑白灰体系
- 开发环境会自动注销 Service Worker 并清理旧缓存
- `sw.js` 在 localhost 下会自我注销并清理旧缓存
- Service Worker 不再 cache-first 缓存 `/_next/` JS/CSS，避免本地开发时加载旧 UI 包导致 hydration mismatch

### 1. 复述训练闭环

- 新增复述训练数据 `retellingPractice`
- 新增复述反馈算法 `createRetellingFeedback`
- 复述反馈支持统计关键点覆盖情况
- 复述反馈支持识别材料高频词使用情况
- 复述反馈会根据覆盖度、长度和关键词给出分数
- 复述反馈会给出下一遍建议
- 练习页新增复述训练模块
- 复述训练支持使用提示句作为支架
- 复述提交后会保存为本地练习记录
- 复述提交后会计入本周输出分钟
- 跟读记录列表会标识练习类型，区分跟读、复述、写作和角色练习
- 新增复述反馈单元测试

### 2. 复述沉淀复习闭环

- 复述训练模块支持保存整段复述句
- 复述训练模块支持保存材料关键词
- 保存复述句会进入词句本
- 保存复述关键词会进入词句本
- 保存复述内容会自动生成多类型复习卡
- 重复保存同一复述句或关键词不会重复生成卡片

### 3. 录音复述

- 复述训练支持麦克风录音
- 复述录音和跟读录音复用同一套浏览器录音能力
- 复述录音支持浏览器转写兜底
- 复述录音会请求云端或本地转写服务
- 复述转写结果会自动填入复述文本框
- 复述转写后会自动生成关键点反馈
- 复述录音会保存为本地练习记录
- 复述录音会计入本周输出分钟

### 4. 功能可用性排查

- 学习页播放按钮已绑定英文朗读
- 学习页慢速、循环和跟读按钮已补齐可用动作
- 练习页模式卡片“进入”按钮已改为可响应入口
- 复习页播放按钮统一使用英文朗读工具
- 材料库和词句本筛选数量已从按钮改为信息展示
- 英文朗读统一优先选择英文语音，并降低语速

## 验收标准

- 练习页可以看到复述训练模块
- 用户可以输入英文复述文本
- 用户可以点击提示句辅助生成复述
- 提交复述后可以看到分数、已覆盖点、待补充点和建议
- 复述记录会进入本地练习记录
- 复述会计入学习进度输出分钟
- 复述反馈算法有单元测试覆盖
- 复述句可以保存为复习卡
- 复述关键词可以保存为复习卡
- 复述训练可以录音
- 复述录音完成后可以转写并生成反馈
- 用户可见的学习页播放、慢速、循环、跟读按钮点击后有明确响应
- 练习页模式入口点击后有明确滚动或未完成提示
