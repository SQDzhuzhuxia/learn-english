# ADR 0001：使用 Next.js 作为 Web/PWA 基础框架

日期：2026-05-27
状态：Accepted

## 背景

项目第一阶段需要快速做出可在电脑和手机浏览器使用的 Web/PWA，同时后续还要支持云同步、AI 代理接口、数据接口、部署和未来开源。

## 决策

使用 Next.js + React + TypeScript 作为第一阶段应用框架。

## 备选方案

- Vite + React：更轻，但需要自己补齐服务端接口、部署约定和后续同步结构。
- Flutter：跨端能力强，但第一阶段验证 Web 学习闭环会更重。
- React Native：适合移动端，但不适合作为最快的 Web/PWA 起点。

## 影响

- 可以先把学习闭环跑通，再决定是否封装移动端。
- API routes 可以承载 AI 代理和云同步接口。
- 项目结构更接近可发布的工程形态。

## 后续动作

- 完成 Next.js 工程骨架。
- 建立主要页面静态原型。
- 在 v0.5 接入 PWA 安装和云同步。
