# 功能排查记录 v0.1

日期：2026-05-30

## 本轮结论

本轮重点排查“UI 已经完成，但部分按钮点击无反应、英文朗读声音不自然”的问题。

问题分为三类：

- 已实现但缺少事件绑定：学习页播放、慢速、循环、跟读按钮。
- UI 看起来像按钮但只是状态展示：材料库和词句本的筛选结果数量。
- 功能仍处于后续计划：真实音频材料播放器、音素级发音评分、AI 增强自由对话。

## 已修复

- 学习页主播放按钮已绑定英文朗读。
- 学习页慢速按钮已绑定慢速英文朗读。
- 学习页循环按钮已支持开启和停止循环慢速朗读。
- 学习页跟读按钮已支持慢速播放并提示进入练习页完成录音反馈。
- 练习页“播放原句”改为统一英文朗读工具。
- 复习页播放按钮改为统一英文朗读工具。
- 练习页四个模式卡片的“进入”按钮已可响应。
- 材料库和词句本的筛选数量从按钮改为信息块，避免用户误以为可以点击。
- 英文朗读统一优先选择英文语音，并降低语速，减少默认系统语音导致的沙哑和不自然。
- 场景口语角色扮演已进入可用闭环，支持三轮对话、反馈、记录和复习沉淀。
- 新增服务端高质量 TTS 接口，配置后会优先使用云端或本地 TTS，未配置时回退浏览器朗读。

## 当前仍未完成

- 真实音频材料播放器：目前主要是 TTS 朗读，不是真实材料自带音频播放和切句。
- 内置离线 TTS 模型：目前支持接入本地 OpenAI-compatible TTS 服务，但还没有把离线模型打包进应用。
- 音素级发音评分：当前跟读评分主要基于转写文本对比，不是专业发音评分。
- AI 增强自由对话：当前角色扮演先使用本地固定场景和规则反馈，还不是开放式 AI 对话。

## 后续建议

下一步应进入“功能可用性稳定 Sprint”：

1. 补齐所有可见按钮的真实动作、跳转、禁用态或未完成提示。
2. 增加统一的 Toast/提示系统，避免用户点击后不知道发生了什么。
3. 增加高质量 TTS 配置界面和语音试听，减少手动改环境变量的成本。
4. 给学习页、练习页和复习页增加最小交互测试。
