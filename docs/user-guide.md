# Learn English 使用教程

这份教程面向本地测试和个人日常使用。目标不是把每个按钮都背下来，而是让你用这个项目形成稳定的英语学习闭环：真实材料输入、逐句理解、输出练习、间隔复习、进度反馈。

## 1. 本地启动

在项目根目录运行：

```bash
npm install
npm run dev
```

启动后打开：

```text
http://localhost:3000
```

如果 `localhost` 访问异常，也可以试：

```text
http://127.0.0.1:3000
```

本地开发时推荐定期跑一次完整质量检查：

```bash
npm run release:check
```

## 2. 第一次使用建议

第一次不要先研究所有功能，按这个最小闭环走一遍：

1. 打开首页，也就是今日页。
2. 选择系统推荐的当前材料。
3. 进入学习页，逐句听、读、看解释。
4. 保存 2 到 3 个真正想用的表达。
5. 去练习页做一次跟读、一次复述或一次短写作。
6. 去复习页完成今天到期的卡片。
7. 去进度页看今天的输入、输出和薄弱项。

只要这个闭环跑通，项目就开始真正服务你的英语学习了。

## 3. 主要页面

项目的核心页面如下：

- `/`：今日页。显示今天建议学什么、练什么、复习什么。
- `/library`：材料库。查看内置材料、课程路径和阶段进度。
- `/library/import`：导入材料。把自己的英文文本导入项目。
- `/study`：学习入口。进入当前材料或材料列表。
- `/study/[id]`：材料学习页。逐句学习、解释、保存表达。
- `/practice`：练习页。跟读、复述、写作、角色扮演和题库练习。
- `/review`：复习页。用间隔重复复习词句本内容。
- `/notebook`：词句本。查看保存过的词、短语和句子。
- `/progress`：进度页。查看学习量、输出量、复习趋势和弱项画像。
- `/settings`：设置页。配置 AI、语音、本地数据和同步相关能力。

## 4. 推荐的每日 30 分钟流程

如果你每天只有 30 分钟，建议按这个节奏：

| 时间 | 做什么 | 目的 |
| --- | --- | --- |
| 5 分钟 | 复习到期卡片 | 防止学过的表达遗忘 |
| 10 分钟 | 学习当前材料 | 获得可理解输入 |
| 5 分钟 | 保存重点表达 | 把材料转成个人词句本 |
| 5 分钟 | 跟读或复述 | 把理解转成口头输出 |
| 5 分钟 | 短写作或角色扮演 | 练习主动表达 |

原则是：每天都要同时有输入、输出和复习。只看材料不输出，提升会很慢；只刷题不复习，遗忘会很快。

## 5. 学习一篇材料

从今日页或材料库进入一篇材料后，学习页会把材料拆成句子。推荐这样使用：

1. 先听当前句，不急着看中文。
2. 自己猜句子大意。
3. 查看当前句解释，确认关键词、语法和语境。
4. 播放或朗读当前句，模仿语音节奏。
5. 对有用的表达点击保存，加入词句本。
6. 继续下一句。
7. 学完后看整篇解释和摘要。

学习时不要把所有陌生词都保存。只保存你未来真的想说、想写、想听懂的表达。

## 6. 导入自己的材料

进入：

```text
/library/import
```

可以导入自己的英文文本。建议材料满足这些条件：

- 难度在 A1 到 A2 附近。
- 句子数量 5 到 12 句为宜。
- 内容来自真实生活，比如租房、看病、购物、工作沟通、学校沟通、移民生活。
- 每篇材料只聚焦一个场景。

导入时通常需要填写标题、类型、等级和正文。项目会把正文拆成句子，后续学习页、今日页和练习页都可以基于这些句子生成学习任务。

如果材料有音频，也可以填写音频地址和切句信息。没有音频也没关系，项目可以使用浏览器 TTS 或本地语音服务朗读。

## 7. 跟读练习

进入：

```text
/practice#practice-shadowing
```

推荐流程：

1. 听一遍目标句。
2. 看文本，跟读一遍。
3. 不看文本，再跟读一遍。
4. 点击录音，说出目标句。
5. 查看转写结果、完整度和发音反馈。
6. 对低分句子重复 2 到 3 次。

跟读的目标不是一次满分，而是让嘴巴熟悉真实句子的节奏、重音和连贯表达。

## 8. 复述练习

进入：

```text
/practice#practice-retelling
```

复述练习会根据当前材料给出要点和开头提示。推荐流程：

1. 先看材料摘要和关键词。
2. 不看原文，用简单英语复述。
3. 提交后查看覆盖了哪些要点。
4. 保存系统建议的自然表达。
5. 对同一篇材料隔天再复述一次。

复述时不要追求复杂句。优先做到意思完整、句子清楚、能持续说下去。

## 9. 短写作练习

进入：

```text
/practice#practice-writing
```

短写作适合练习把刚学到的表达主动用出来。推荐流程：

1. 阅读题目和提示。
2. 写 1 到 3 个简单句。
3. 尽量使用当前材料里的表达。
4. 提交后查看 AI 批改或本地降级反馈。
5. 保存修改后的好句子。

短写作的重点不是写长，而是写得准确、自然、可复用。

## 10. 角色扮演练习

进入：

```text
/practice#practice-roleplay
```

角色扮演用于模拟真实场景，比如看医生、租房、购物、工作沟通。推荐流程：

1. 先阅读对方说的话。
2. 自己用英文回答。
3. 不会说时再看建议回复。
4. 提交后继续下一轮。
5. 练完后查看总结、目标完成度和可改进表达。

如果启用了 AI，项目可以生成追问、会话总结和更自然的替代表达。如果没有配置 AI，也会使用本地规则提供基础反馈。

## 11. 题库练习

练习页还可以把材料转成题库任务，例如识别、填空、选择、短答和句子输出。推荐用法：

1. 先学完一篇材料。
2. 基于这篇材料生成练习题。
3. 当天做一部分，不要一次刷完。
4. 错题进入后续复习。
5. 过几天再做同主题变体题。

这个项目的核心思路是“一份真实材料生成多种练习”，而不是靠手写大量固定题目。

## 12. 复习词句本

进入：

```text
/review
```

复习页会把词句本里的内容按间隔重复调度。推荐流程：

1. 先完成“到期”卡片。
2. 看正面时先自己回忆，不要马上翻答案。
3. 翻答案后按真实掌握程度评分。
4. 对困难卡片可以回炉或暂停。
5. 对已经掌握的卡片让系统拉长间隔。

评分建议：

- 忘了：完全想不起来。
- 困难：想起来了，但很慢或不确定。
- 一般：基本会，但还不稳。
- 简单：能快速理解或说出来。

## 13. 查看进度和弱项

进入：

```text
/progress
```

重点看这些信息：

- 输入量：你学了多少材料、句子和分钟数。
- 输出量：你做了多少跟读、复述、写作和角色扮演。
- 复习趋势：到期卡片是否堆积。
- 输入输出平衡：是否只看不说、只练不复习。
- 长期弱项画像：哪些场景、表达类型或输出能力最薄弱。
- 场景地图：哪些生活场景练得多，哪些还空白。

如果进度页提示某类能力薄弱，下一天优先安排对应练习。例如口语输出少，就先做跟读和角色扮演；写作弱，就做短写作。

## 14. AI 配置

项目不强制依赖 AI。没有 AI 配置时，仍然可以使用本地规则和降级解释。AI 是增强项，不是启动项目的前提。

如果要启用 AI，可以在项目根目录创建 `.env.local`，参考 `.env.example`：

```bash
AI_PROVIDER=openai-compatible
AI_BASE_URL=http://127.0.0.1:11434/v1
AI_MODEL=your-model-name
AI_API_KEY=your-api-key
```

实际变量以 `.env.example` 为准。修改环境变量后需要重启开发服务：

```bash
npm run dev
```

AI 能增强这些能力：

- 当前句解释。
- 整篇材料解释。
- 写作批改。
- 角色扮演追问。
- 会话总结。
- 练习题生成。

## 15. 本地语音配置

浏览器自带 TTS 和语音识别能力时，很多功能可以直接使用。如果要接入更完整的本地语音运行时，可以先检查配置：

```bash
npm run speech:check
```

本地开发运行时自检：

```bash
npm run speech:dev-runtime -- --self-test --json
```

Windows 本地运行时自检：

```bash
npm run speech:windows-runtime -- --self-test --json
```

如果要启用本地发音评分服务，可以在 `.env.local` 中配置：

```bash
PRONUNCIATION_PROVIDER=local
PRONUNCIATION_BASE_URL=http://127.0.0.1:8090
```

如果本地服务没有启动，项目会显示服务不可用或使用降级反馈。真正的生产级音素强制对齐需要外部模型服务支持，Web 项目本身只负责接入、诊断和展示结果。

## 16. 数据备份

本地测试时，学习数据主要保存在浏览器本地存储中。建议定期在设置页导出备份，尤其是在你开始大量导入材料和积累词句本之后。

建议习惯：

- 每周导出一次本地备份。
- 大量导入材料前先备份。
- 清理浏览器数据前先备份。
- 换浏览器或换电脑时用导入功能恢复。

Supabase 同步是可选能力。本地测试阶段可以先不用发布和云同步，把核心学习闭环跑顺更重要。

## 17. 一周使用计划

如果你想认真验证这个项目是否能帮助你提高英语，可以按下面计划用一周：

| 天数 | 任务 |
| --- | --- |
| 第 1 天 | 选择一条课程路径，学完第一篇材料，保存 3 个表达 |
| 第 2 天 | 复习到期卡片，对第一篇材料做跟读 |
| 第 3 天 | 学第二篇材料，对第一篇材料做复述 |
| 第 4 天 | 对第二篇材料做短写作，保存修改后的句子 |
| 第 5 天 | 做一个角色扮演场景，记录不会说的表达 |
| 第 6 天 | 导入一篇自己的真实材料，并完成逐句学习 |
| 第 7 天 | 查看进度页，找出下周最需要补的弱项 |

一周后重点看三个问题：

1. 你是否每天都有复习。
2. 你是否能把材料里的表达说出来或写出来。
3. 你是否知道自己下一步最该补什么。

## 18. 常见问题

### 没有 AI 还能用吗？

可以。项目会使用本地规则和降级解释。AI 是增强项，不是启动项目的前提。

### 没有声音怎么办？

先检查浏览器是否允许播放声音和麦克风权限。再确认系统音量、输入设备和浏览器权限。如果使用本地语音服务，运行 `npm run speech:check` 查看配置。

### 录音没有转写怎么办？

检查浏览器是否支持语音识别、麦克风权限是否打开。本地运行时没有启动时，语音相关能力会降级。

### 内容太难怎么办？

换更低等级的材料，或者把自己的材料拆短。A1 到 A2 阶段最重要的是高频、清楚、可复用，不要一开始就使用长文章。

### 复习卡太多怎么办？

先完成到期卡片，不要继续无节制保存新表达。对暂时不重要的卡片可以暂停。每天保存少量高价值表达，比保存大量低价值生词更有效。

### 学了很多但还是不会说怎么办？

通常是输出练习不足。每天至少做一次跟读或复述，把看懂的内容转成嘴巴能说出来的句子。

## 19. 判断项目是否跑通

本地测试阶段，可以用下面标准判断项目是否已经对你有实际帮助：

- 能从今日页进入一篇当前材料。
- 能逐句听、读、看解释。
- 能保存重点表达到词句本。
- 能完成一次跟读、复述或短写作。
- 能在复习页看到并复习保存的内容。
- 能在进度页看到学习和输出记录。
- 能根据弱项提示决定明天练什么。

如果这些都跑通，这个项目的核心学习闭环就成立了。后续要继续提升效果，重点不是堆更多页面，而是持续增加真实材料、加强输出反馈、让复习和弱项诊断更准确。
